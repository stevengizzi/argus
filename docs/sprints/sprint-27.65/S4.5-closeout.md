## Judgment Calls

1. **Zero-R guard placement**: Placed in `_build_signal()` after target computation rather than in `_handle_testing_level()`. This is closest to where the targets are computed and matches the pattern_strategy.py approach (guard near signal construction). The orb_base.py approach (guard after signal construction) would also work but would waste the pattern strength computation.

2. **Concurrent position check placement**: Placed after "all 5 conditions passed" telemetry and before `_build_signal()`. This means telemetry records that all entry conditions passed even if the concurrent check blocks the signal — which is useful observability (shows the strategy *wanted* to trade but was limited).

3. **AccountUpdateEvent Option A**: Chose the event bus path over removing the event class. This is more consistent with the project's architecture (all events through the bus) and makes the AccountUpdateEvent actually useful for future subscribers (e.g., analytics, logging).
