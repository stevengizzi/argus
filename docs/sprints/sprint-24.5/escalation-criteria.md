# Sprint 24.5 — Escalation Criteria

These conditions require halting the current session and escalating to the work
journal (human-in-the-loop) or sprint planner (autonomous mode) for resolution.

## Critical Escalations (Halt Immediately)

1. **Strategy on_candle() behavior change:** If instrumentation inadvertently
   alters the return value of `on_candle()` for any strategy (e.g., a
   `record_evaluation()` call that short-circuits before the return statement),
   halt and fix before proceeding. Detection: any existing strategy test that
   previously passed now fails.

2. **Ring buffer blocks candle processing:** If profiling or observation shows
   `record_evaluation()` adding measurable latency (>100μs) to the candle
   processing hot path, halt and investigate. The deque append should be
   sub-microsecond; if it isn't, something is wrong.

3. **BaseStrategy construction breaks:** If adding `_eval_buffer` to
   `BaseStrategy.__init__()` causes any existing strategy construction test to
   fail, halt. This suggests an `__init__` signature or super().__init__()
   issue that must be resolved before continuing.

4. **Existing REST endpoints break:** If the strategies router changes cause any
   existing strategy listing or detail endpoint to return different results, halt.
   New endpoints must be additive only.

## Significant Escalations (Complete Current Task, Then Escalate)

5. **SQLite write throughput insufficient:** If persistence writes cannot keep up
   with event generation rate (>200 events/sec), escalate to evaluate batching
   strategy. May need write-behind with periodic flush instead of inline writes.

6. **Frontend 3-column layout disruption:** If the slide-out panel from
   StrategyOperationsCard disrupts the Section 4 three-column layout
   (DecisionTimeline + CatalystAlerts + RecentSignals), escalate for design
   adjustment. The Decision Stream must not displace existing Orchestrator
   page content.

7. **Test count deviation >50%:** If any session requires significantly more or
   fewer tests than estimated (e.g., S1 needs 20+ tests instead of ~10),
   escalate to reassess session scoping and compaction risk.

8. **AI Insight clock bug is not in summary.py:** If the elapsed session time
   calculation is not in `_assemble_insight_data()` but is instead generated by
   the AI model from uptime data, the fix approach changes from code fix to
   prompt engineering. Escalate for reassessment.

## Informational (Log and Continue)

9. **Pre-existing test failures:** DEF-048 (4 xdist failures) and DEF-049
   (orchestrator isolation failure) are known pre-existing. Log their presence
   but do not halt.

10. **Strategy-specific instrumentation ambiguity:** If it's unclear exactly
    where in a strategy's code path an evaluation event should be emitted (e.g.,
    nested conditionals with multiple possible exit points), use best judgment
    and document the choice. This is instrumentation, not logic change.
