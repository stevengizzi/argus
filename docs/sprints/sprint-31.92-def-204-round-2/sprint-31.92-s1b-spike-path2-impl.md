Increment `case_a_count`. Track the order's pending duration; cancel after `case_a_max_age_seconds` configurable window (default 30 seconds for the spike — beyond that, the trial is recorded as case A with `case_a_max_age_seconds_observed = max_observed`).
     - **Release event:** if a case-A held order eventually fills (locate-pending-borrow released), record the elapsed time from order submission to fill as a release-event observation. These observations populate the release timing distribution (p50, p95, p99, max).
   - **Substring fingerprint extraction:** for case-B observations, extract the substring most likely to be stable across trials and IBKR Gateway versions. The default candidate is `"contract is not available for short sale"` per H5; if observed strings disagree, capture the longest common substring across trials and surface in close-out for S3a fingerprint-broadening decision (regex or substring-list).

### Verification Grep Precision

When kickoffs include verification grep commands, prefer the more precise patterns:

- **Section counting:** use `^## [1-9]\.` (anchor + literal dot) rather than `^## [0-9]` to avoid double-counting.
- **Human-authored content with TitleCase:** use `grep -i` for content like Markdown bold-list section names. Lowercase patterns will return false negatives against TitleCase source.
- **Token-presence checks across rejection-framed content:** when checking that a rejected pattern is absent, scan only validation logic, not docstrings/rationale blocks. A rejection rationale that names the token is not a reintroduction.
