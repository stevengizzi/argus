## Documentation automation

The audit campaign runs 22 FIX sessions across 8 stages. Manual tracking is explicitly out of scope. Instead:

**Each FIX session writes a structured JSON appendix** to `docs/audits/audit-2026-04-21/session-logs/{FIX-NN-name}.json` as part of its commit, per the instruction block in each prompt. The schema includes session ID, stage, safety tag, start/end timestamps, duration, findings addressed/resolved/verified-only/promoted-to-DEF, test counts (baseline, post, net delta, new failures), commit SHAs, DEFs closed/opened, DECs opened, Tier 2 review outcome, and anomalies (free text).

**The automation script** at `scripts/audit_2026_04_21_status.py` consumes these JSON files and:
- Regenerates `docs/audits/audit-2026-04-21/STAGE-LOG.md` (full per-stage verdict tables)
- Updates `CLAUDE.md`'s "Current State" Tests line
- At Stage 8 end, with `--closeout`, generates `CLOSEOUT.md` skeleton for the Work Journal conversation to flesh in

Operator commands:
```bash
# After each session commits (or at any stage boundary):
python scripts/audit_2026_04_21_status.py

# Campaign progress at a glance:
python scripts/audit_2026_04_21_status.py --status

# At Stage 8 end:
python scripts/audit_2026_04_21_status.py --closeout
```

The script is idempotent. Safe to run after every commit, at stage boundaries, or once at campaign end.
