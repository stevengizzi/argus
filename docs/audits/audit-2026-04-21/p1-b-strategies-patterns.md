## Summary

Read-only audit of every strategy file under `argus/strategies/` plus the 10 pattern modules under `argus/strategies/patterns/`. Confirms the project-knowledge note that DEF-138 covers 15 strategies (none currently wire the window-summary infrastructure). Surfaces one state-leak bug in `VwapBouncePattern`, a wholly-dead `StrategyConfig.allowed_regimes` YAML field (overridden by hardcoded Python lists in every strategy), and several convention inconsistencies across the 15-strategy surface. No critical regressions in signal or risk paths.

## Positive Observations

- **DEC-261 mutual exclusion** at [argus/strategies/orb_base.py:102-107, 607-618](argus/strategies/orb_base.py#L102-L107) is implemented cleanly with a `ClassVar` set, a runtime opt-out flag, and explicit cleanup in `reset_daily_state`. Pattern worth replicating if future strategy families need same-symbol exclusion.
- **PatternBasedStrategy wrapper** at [argus/strategies/pattern_strategy.py](argus/strategies/pattern_strategy.py) is a genuinely good abstraction ‚Äî pattern modules stay pure; the wrapper owns operating-window enforcement, per-symbol candle windows, backfill from `IntradayCandleStore`, and SignalEvent assembly. Adding a new pattern now requires only pattern detection logic + a Pydantic config.
- **PatternParam structured metadata** (Sprint 29 S1‚ÄìS2 retrofit) is a solid foundation for BacktestEngine sweep grid generation, Pattern Library UI, and fingerprint introspection. The factory's `extract_detection_params()` uses it to avoid hardcoding parameter names ‚Äî clean design.
- **Lazy-import factory with class cache** at [argus/strategies/patterns/factory.py:99-130](argus/strategies/patterns/factory.py#L99-L130) ‚Äî pattern modules are only imported when first requested, and the cache prevents repeated `importlib` overhead. Startup-time-friendly.
- **Parameter fingerprint with exit-override backward-compat** at [argus/strategies/patterns/factory.py:206-259](argus/strategies/patterns/factory.py#L206-L259) ‚Äî the `if exit_overrides:` branch preserves hash stability for detection-only variants while distinguishing exit-varied variants. Good migration pattern.
- **Shadow-mode routing centralized** at [argus/main.py:1700-1715](argus/main.py#L1700-L1715) ‚Äî strategies emit `SignalEvent`; main.py decides whether to execute or route to the `CounterfactualTracker`. Strategies stay mode-agnostic. Clean separation.
- **Telemetry fire-and-forget** at [argus/strategies/base_strategy.py:436-466](argus/strategies/base_strategy.py#L436-L466) ‚Äî broad `except Exception: pass` is the correct design here ("Telemetry must never impact strategy operation"). Aligns with the DEF-103 yfinance / VIXDataService staleness pattern.
- **Session-state decoupling** (DEC-028) is consistently honored across all 15 strategies ‚Äî each has `reset_daily_state()` that clears symbol state but not config/allocation.
- **DEC-061 UTC‚ÜíET conversion** is universally applied (`astimezone(ET).time()`) ‚Äî no regressions from the original bug.
- **Sprint 24 S6a `share_count=0` contract** is consistent across all 15 strategies ‚Äî every signal-build path sets `share_count=0` so the Quality Engine or legacy sizer fills it in.
- **`_has_zero_r` helper on BaseStrategy** is the right place for the guard (central, inherited by all 15). Just needs two more call sites (M6, M7).
- **Pattern scoring inline comments** ‚Äî most patterns label each scoring component with `# Name (0-N): description`. Reads like a score spec, not like code. Worth preserving as a convention.
- **Backfill from `IntradayCandleStore`** at [argus/strategies/pattern_strategy.py:176-242](argus/strategies/pattern_strategy.py#L176-L242) lets pattern strategies evaluate immediately on first live candle instead of waiting for full `lookback_bars` ‚Äî important for mid-session restarts (DEF-064 mitigation).
