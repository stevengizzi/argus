## Positive Observations

1. **Single-source-of-truth fill model.** `argus/core/fill_model.py:evaluate_bar_exit()` is a pure function used by BacktestEngine, CounterfactualTracker, and their tests — no code duplication, no drift. Exemplary design worth replicating wherever live / backtest parity matters.
2. **AMD-7 bar ordering documented inline.** `counterfactual.py:464-477` docstring explicitly enumerates the three-step ordering (compute effective stop → evaluate exit → update state for next bar) so future contributors can't silently break parity with the BacktestEngine.
3. **Abstract `CatalystSource` ABC.** Three sources with uniform lifecycle (`start` / `fetch_catalysts` / `stop` / `source_name`) — clean, testable, and made firehose support a per-source implementation detail rather than a pipeline refactor.
4. **Rate-limited warning pattern.** `CounterfactualStore._warn` (60s interval, `counterfactual_store.py:471-479`) and `CatalystStorage` shared error-quiet-period patterns. Prevents log flooding during downstream DB outages without hiding the incident. Same pattern as `EvaluationEventStore` — consistent.
5. **Two-layer dedup.** Cross-source hash dedup (`__init__.py:182-188`) + per-(symbol,category,window) semantic dedup (`__init__.py:259-317`). Complementary, not redundant — the first eliminates *exact* duplicate headlines across sources; the second collapses *clustered* catalysts about the same event.
6. **Batch-then-publish.** `store_catalysts_batch` in a single transaction, then publish events out of band with per-item `try/except` (DEC-312). Prevents half-committed DB state on publish failure.
7. **Graceful degradation at every Claude boundary.** Classifier and BriefingGenerator both have tested fallback paths (keyword classifier + no-Claude markdown brief). The system continues producing (lower-quality) output when Claude is unavailable, rather than failing.
8. **Strong config validation.** `QualityWeightsConfig.validate_weight_sum` (sum-to-1.0 with tolerance), `QualityThresholdsConfig.validate_descending` (strictly descending in [0,100]), `QualityRiskTiersConfig.validate_tiers` (min≤max, both in [0,1]) — Pydantic `model_validator` keeps invalid configs from ever loading.
9. **Separate SQLite DBs per subsystem** (DEC-345 pattern): `catalyst.db`, `counterfactual.db`, `argus.db`, `evaluation.db`, `learning.db`. Avoids WAL contention. (Ironically C1 is a bug *within* this pattern, not against it.)
10. **Counterfactual persistence is non-blocking.** `asyncio.get_running_loop().create_task(store.write_*)` inside `track()` and `_close_position()` ensures DB writes never block the tracking hot path. Caught `RuntimeError` if no loop (tests) — nice defensive detail.
11. **Explicit file-by-file `# type: ignore` usage** — when `type: ignore` is used it's narrow and on the specific line. No wholesale `# type: ignore[misc]` file-level escapes.
12. **VIX context is genuinely optional.** `BriefingGenerator._build_vix_context` checks three layers (service present / ready / non-stale data) before emitting — no silent inclusion of stale VIX readings into the brief.
