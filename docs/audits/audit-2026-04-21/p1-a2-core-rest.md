## Positive Observations

1. **`exit_math.py` + `fill_model.py` are model library code.** Pure functions, no I/O, no argus imports, explicit pre/post-condition docstrings, `StopToLevel` + `FillExitReason` StrEnums for readability. Worst-case-for-longs priority (stop > target > time-stop > EOD) codified in a single 40-line function used by BacktestEngine, CounterfactualTracker, and Order Manager. This pattern — extract decision math into a pure lib, import into three consumers — is worth replicating elsewhere (particularly in `quality_engine.py`, which still mixes math with DB access).

2. **`market_calendar.py` cross-validates against `ui/src/utils/marketTime.ts`.** Python + frontend both implement the Anonymous Gregorian algorithm so holiday detection is identical on both sides of the API. This parity is noted in the module docstring. Exemplary for any future shared logic (market hours, session boundaries, etc.).

3. **`RegimeHistoryStore` idempotent schema migration.** `_migrate_add_vix_close()` (Sprint 27.9) checks `PRAGMA table_info` before `ALTER TABLE`, so the Sprint 27.9 VIX addition rolled out with zero operator intervention. `_CREATE_TABLE_SQL` + `CREATE INDEX IF NOT EXISTS` means `.initialize()` is fully idempotent.

4. **`RegimeClassifierV2` cleanly delegates V1 for `classify()`/`compute_indicators()`.** No re-implementation; V2 is strictly additive (8 calculators → 7 new fields). When V1 thresholds change, V2 picks them up automatically. This is what the docstring promised and the code delivers.

5. **`Orchestrator.latest_regime_vector_summary` duck-type pattern.** Despite its typing cost (L8), the `hasattr(x, "to_dict")` gate was a deliberate Sprint 27.6 Session 6 choice to avoid circular imports (`Orchestrator → RegimeVector → maybe Orchestrator`). A reasonable trade-off that's still documented inline.

6. **`PDTTracker._business_days_ago` is a 10-line self-contained helper.** No external deps, no library (doesn't pull `pandas.tseries.offsets.BDay`). For risk-critical code, the minimal-dependency instinct here is correct.

7. **`EventBus._safe_call` error isolation.** One bad handler can't break other subscribers — exception is logged with seq + handler qualname, doesn't propagate. This is the right behaviour for a pub/sub hub.

8. **`ThrottleAction` three-state worst-wins pattern.** `SUSPEND > REDUCE > NONE` ordered explicitly in `PerformanceThrottler.check()` (lines 126-130). Easy to read, hard to get wrong.
