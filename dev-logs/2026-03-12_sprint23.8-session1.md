# Sprint 23.8 Session 1 — Pipeline Resilience + Symbol Scope

**Date:** 2026-03-12
**Sprint:** 23.8
**Tests:** 2,516 passing (5 new)

## Summary

Made the intelligence polling loop resilient to crashes and hangs, fixed
the symbol scope from full universe to scanner watchlist, and ensured the
polling task is properly monitored.

## Changes

1. **startup.py** — Wrapped `pipeline.run_poll()` with `asyncio.wait_for(timeout=120s)`.
   Added `asyncio.TimeoutError` handler that logs CRITICAL and continues the loop.
   Added per-cycle log line showing symbol count and first 5 symbols. Changed QA debug
   log ("Polling loop coroutine entered") from INFO to DEBUG.

2. **server.py** — Cleaned up `get_symbols()` to match spec: prioritizes scanner watchlist,
   falls back to capped viable universe, returns `[]`. Removed inline logging from
   `get_symbols()` (now in startup.py polling loop). `_poll_task_done` callback and task
   reference on `app_state` were already in place from QA session.

3. **Config alignment** — Updated `test_config.py` assertions from `enabled is False` to
   `enabled is True` to match `system.yaml` (changed during QA). Updated comment in
   `system.yaml` to reflect enabled state.
